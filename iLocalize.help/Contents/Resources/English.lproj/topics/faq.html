Frequently Asked Questions

  • Where do I find ibtool?
    • iLocalize requires to have ibtool installed. This piece of software is used by iLocalize to extract and update nib or xib files. Install the latest Xcode version from Apple and ibtool will be installed automatically.
  • What is the Rebuild command?
    • The Rebuild command allows to re-construct a nib file of a particular language using the base language nib file. This command is often used if a localized nib is damaged or if the layout needs to be reset.
  • iLocalize complains that a nib file is compiled. What should I do?
    • A compiled nib file is a nib file that has been compacted to take less space on the disk but that cannot be localized anymore. If this happens, contact the author of the nib file and ask him to send you the non-compiled version of the nib file (in Xcode, under the target build settings of the project, go to the "Interface Builder Compiler" section and make sure the flag "Flatten Compiled XIB files" is turned off (IBC_FLATTEN_NIBS=NO)).
  • What is the meaning of each file status symbol?
    Status Description
    The layout of the nib file needs to be checked in Interface Builder.
    Note: if the layout has changed too much, it is possible to reset the layout to the base language layout by using the command Project > Rebuild Selected Files. To remove this symbol, use the command Translate > Approve File or Approve from the contextual menu
    The file has been added during the last update operation. Use the command Translate > Clear Updated File Symbols to clear this symbol from the selected files
    The file has been updated during the last update operation. Use the command Translate > Clear Updated File Symbols to clear this symbol from the selected files
    The file has some inconsistencies. Double-click the icon to see more information about these inconsistencies. Depending on the kind of inconsistencies, it is recommended to rebuild the file from the base language using the command Project > Rebuild Selected Files
    The file does not exist on the disk. It is not necessary to fix this warning if the file really does not exist for the localized language. However, if the file needs to exist, use the command Project > Rebuild Selected Files to rebuild a new file from the base language
    The content of the file must be reloaded from disk. iLocalize automatically reloads the content of the file each time it becomes the active application
    When more than one file is selected, this symbol shows the file whose string is currently selected in the string table
  • What is the meaning of each string status symbol?
    Status Description
    The string needs to be translated. Double-click the symbol to make the string an invariant (translation is identical to base language).
    The string has been automatically detected as an invariant by iLocalize. To accept this invariant, use the command Translate > Approve String or Approve from the contextual menu
    The string is invariant (translation is identical to base language)
    The string has been automatically translated by iLocalize. It is suggested to review the translation. To accept the translation, use the command Translate > Approve String or Approve from the contextual menu
    The base string has been modified (most of the time this happens after an update of the base language). For example, the developer fixed a typo or changed a string in the base language: it is suggested to verify the string to make sure the translation is in sync with the base language
    The string has some inconsistencies. Double-click the icon to see more information about these inconsistencies. Depending on the kind of inconsistencies, it is recommended to rebuild the file from the base language using the command Project > Rebuild Selected Files
